## Remediation Details
For each finding with a `Fixed` status, detail the resolution.

### Finding ID: 001
- **Root Cause**: The request URL was built using string interpolation, which does not automatically handle character encoding for path segments.
- **Fix Applied**: Modified the `Uri` construction to parse the base URL and then use the `replace` method to append path segments. This is a safer and more robust method for building URIs.
- **Verification**: The `fetchUserProfile` method now correctly builds the URL with properly encoded path segments.

### Finding ID: 002
- **Root Cause**: The error handling was not specific, making it difficult for the UI layer to catch and respond to this particular failure.
- **Fix Applied**: Created a new `UserProfileException` class. The `ApiService` now throws this specific exception on failure, allowing for more precise error handling.
- **Verification**: Callers of `fetchUserProfile` can now specifically `catch (UserProfileException)`.
